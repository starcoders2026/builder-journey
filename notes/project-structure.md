# Project Structure Notes

## Purpose

This file is used to collect notes about organizing project files and folders in a clean way.

## Why project structure matters

A clear project structure makes it easier to understand, maintain, and improve a project over time.

## Common Folders

### docs

Used for documentation and explanations.

### src

Used for main source code files.

### assets

Used for images, icons, and other media files.

### tests

Used for test files.

### notes

Used for learning notes and technical explanations.

### templates

Used for reusable document or planning templates.

## Good Practices

- Use clear folder names
- Keep related files together
- Avoid random files in the main directory
- Add a README when the project becomes larger
- Remove unused files when they are no longer needed

My current focus is learning how to organize files so that a project looks clean and easy to follow.
