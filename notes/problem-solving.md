# Problem Solving Notes

## Purpose

This file is used to collect notes about solving technical problems in a clear and structured way.

## Basic Problem Solving Method

1. Understand the problem
2. Read the error or requirement carefully
3. Break the problem into smaller parts
4. Test one possible solution
5. Check the result
6. Improve the solution if needed
7. Write down what worked

## Useful Questions

- What exactly is not working?
- When did the problem start?
- Which file or feature is related to the problem?
- Is there an error message?
- What did I try already?
- What changed after the last working version?

## Good Habits

- Do not change too many things at once
- Keep notes about solved problems
- Test small changes
- Read documentation when needed
- Compare working and broken versions
