# GitHub Notes

## Purpose

This file is used to collect notes about GitHub usage, repository management, and clean commit habits.

## Basic Concepts

### Repository

A repository is a project folder stored on GitHub.

### Commit

A commit is a saved change in the project history.

### Branch

A branch is a separate version of the project used for development or testing.

### README

A README file explains what the project is about and how it is organized.

## Good Commit Habits

- Use clear commit messages
- Keep commits focused on one change
- Avoid random or empty changes
- Update documentation when needed
- Keep the repository structure clean
