# Debugging Notes

## Purpose

This file is used to collect notes about finding, understanding, and fixing problems in software projects.

## What is debugging?

Debugging is the process of finding errors in a project and fixing them step by step.

## Common Debugging Areas

- Syntax errors
- Broken file paths
- Wrong variable names
- Missing dependencies
- API request problems
- Console errors
- Deployment issues

## Basic Debugging Workflow

1. Read the error message carefully
2. Identify where the problem happens
3. Check the related file or function
4. Test one small change
5. Run the project again
6. Repeat until the issue is fixed
