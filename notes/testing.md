# Testing Notes

## Purpose

This file is used to collect notes about testing projects, checking features, and preventing simple mistakes.

## What is testing?

Testing means checking if a project works as expected before sharing or publishing it.

## Main Testing Areas

- Button behavior
- Form inputs
- Page links
- API responses
- Error messages
- Mobile layout
- Browser compatibility
- Deployment checks

## Basic Testing Workflow

1. Decide what feature needs to be tested
2. Try the expected normal behavior
3. Try possible wrong inputs
4. Check error messages
5. Fix the issue if something breaks
6. Test again after the fix
