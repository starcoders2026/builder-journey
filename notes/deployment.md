# Deployment Notes

## Purpose

This file is used to collect notes about publishing projects, hosting options, and deployment workflows.

## What is deployment?

Deployment means making a project available for others to use online.

## Main Topics

- Hosting platforms
- Environment variables
- Build process
- Domain setup
- Error logs
- Version control
- Testing before release

## Basic Deployment Workflow

1. Finish the main project changes
2. Test the project locally
3. Push the latest changes to GitHub
4. Connect the repository to a hosting platform
5. Configure required settings
6. Deploy the project
7. Test the live version
